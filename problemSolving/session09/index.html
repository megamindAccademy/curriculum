What is a Bug? üêõ

A mistake or error in the code that stops it from running or makes it do the wrong thing.

Fun Fact! üí°

It's named after a real moth found inside an early computer in 1947 by Grace Hopper!

Real-Life Scenario: The Broken Game! üéÆ

Imagine you are playing your favorite video game. You jump to catch a coin, but suddenly, your character falls through the floor into an empty black screen! Or you win a level, but your score goes DOWN to zero! üò±

That is exactly what a BUG is! A mistake in the game's code that causes it to act weird.

The 3 Main Types of Bugs üî¨

1

Syntax Error

Like bad grammar in English. E.g., missing a quote or colon.

print("Hello)
2

Runtime Error

Code makes sense but crashes while running. E.g., dividing by zero.

print(10 / 0)
3

Logical Error

The code runs, but gives the wrong answer. E.g., adding instead of subtracting.

print(5 + 2) # Expected 3

Spot the Bug! üîé

Look closely at these code snippets. Can you spot the sneaky bugs hiding in them? Click to reveal!

Snippet 1:

score = 100

print(Score)

NameError (Logical/Runtime)!
Python is case-sensitive. Score (capital S) is completely different from score (small s).

Snippet 2:

for i in range(5)

print("Hello!")

Syntax Error!
Missing the magical colon : at the end of the `for` loop line!
